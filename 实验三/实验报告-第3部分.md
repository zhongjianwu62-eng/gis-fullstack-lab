## 三、技术架构

### 3.1 MapLibre GL JS vs Leaflet 对比

| 特性 | Leaflet 1.9 | MapLibre GL JS 4.7 |
|------|:----------:|:-----------------:|
| 渲染引擎 | Canvas/SVG | **WebGL** |
| 瓦片类型 | 栅格瓦片 (PNG) | **矢量瓦片 (MVT)** |
| 样式修改 | 有限（需换瓦片源） | **完全灵活（客户端实时渲染）** |
| 数据量级 | 万级 | **百万级** |
| 3D 能力 | 无 | **fill-extrusion + terrain** |
| 点聚合 | 需插件 | **内置 cluster** |
| 包体积 | ~42KB | ~220KB |

### 3.2 Style JSON 结构

MapLibre 使用 **Style Specification**（样式规范 JSON）定义地图外观：

```json
{
  "version": 8,
  "sources": { ... },     // 数据源（栅格瓦片/矢量瓦片/GeoJSON）
  "layers": [ ... ],      // 图层（渲染顺序 = 数组顺序）
  "sprite": "...",        // 图标精灵（矢量瓦片用）
  "glyphs": "..."         // 字体（矢量瓦片用）
}
```

## 四、功能实现详解

### 4.1 MapLibre 地图初始化 + Style JSON

**功能描述**：使用 MapLibre GL JS 4.7.1 创建地图实例，通过 Style JSON 定义 5 套底图。

**5套底图方案**：

| 样式 ID | 名称 | 数据源 | 瓦片 URL |
|---------|------|--------|---------|
| osm-bright | OSM 明亮 | OpenStreetMap | `tile.openstreetmap.org/{z}/{x}/{y}.png` |
| carto-dark | 暗黑模式 | CartoDB Dark | `a.basemaps.cartocdn.com/dark_all/...` |
| esri-sat | 卫星影像 | ESRI Imagery | `server.arcgisonline.com/.../World_Imagery/...` |
| carto-light | 浅色简约 | CartoDB Light | `a.basemaps.cartocdn.com/light_nolabels/...` |
| topo | 地形地图 | OpenTopoMap | `a.tile.opentopomap.org/{z}/{x}/{y}.png` |

**核心代码**：

```javascript
const map = new maplibregl.Map({
    container: 'map',
    style: {
        version: 8,
        sources: {
            'osm': {
                type: 'raster',
                tiles: ['https://tile.openstreetmap.org/{z}/{x}/{y}.png'],
                tileSize: 256,
                attribution: '© OSM'
            }
        },
        layers: [{ id: 'osm', type: 'raster', source: 'osm' }]
    },
    center: [116.40, 39.92],
    zoom: 11
});
```

### 4.2 数据驱动样式（第11讲练习1）

**功能描述**：使用 MapLibre 表达式（Expressions）实现数据驱动的颜色、大小、透明度。

**数据源**：北京 32+ 个地标 GeoJSON（7 个分类：landmark/park/school/stadium/shopping/culture/transport）

**数据驱动规则**：

| 视觉变量 | 驱动属性 | 表达式类型 | 效果 |
|---------|---------|-----------|------|
| 圆半径 | `importance` (55~100) | `interpolate linear` | 越重要越大 |
| 圆颜色 | `category` (7类) | `match` | 不同颜色 |
| 文字大小 | `importance` | `interpolate linear` | 越重要字越大 |

**核心代码**：

```javascript
// 数据驱动样式 — 颜色按分类匹配
'circle-color': [
    'match', ['get', 'category'],
    'landmark',  '#e74c3c',   // 历史地标 → 红色
    'park',      '#27ae60',   // 公园 → 绿色
    'school',    '#3498db',   // 高校 → 蓝色
    'stadium',   '#f39c12',   // 体育馆 → 橙色
    'shopping',  '#9b59b6',   // 商圈 → 紫色
    'culture',   '#e67e22',   // 文化 → 棕色
    'transport', '#1abc9c',   // 交通 → 青色
    '#95a5a6'                 // 默认 → 灰色
],
// 数据驱动样式 — 半径按重要性插值
'circle-radius': [
    'interpolate', ['linear'],
    ['get', 'importance'],
    55, 6,    // importance=55 → radius=6px
    100, 18   // importance=100 → radius=18px
]
```

**表达式类型总结**：

| 表达式 | 用途 | 示例 |
|--------|------|------|
| `match` | 精确匹配（枚举映射） | 分类→颜色 |
| `interpolate` | 线性插值（连续映射） | 数值→大小 |
| `step` | 阶梯分段 | 数量范围→颜色 |
| `case` | 条件判断 | 是否换乘站→样式 |
| `get` | 获取属性值 | `['get', 'name']` |
| `concat` | 字符串拼接 | 名称+高度 |

### 4.3 样式切换器（第11讲练习2）

**功能描述**：5 套底图/配色方案，一键切换，保持视角不变。

**切换流程**：

```
用户点击样式
    │
    ▼
保存当前视角 (center, zoom, pitch, bearing)
    │
    ▼
显示 loading 动画
    │
    ▼
map.setStyle(newStyle)  ─── 加载新瓦片
    │
    ▼
map.once('style.load')  ─── 等待完成
    │
    ▼
map.jumpTo(保存的视角)  ─── 恢复视角
    │
    ▼
重建自定义图层 (sources + layers)
    │
    ▼
隐藏 loading，更新 UI
```

**关键代码**：

```javascript
function switchStyle(styleId) {
    const center = map.getCenter(), zoom = map.getZoom();
    const pitch = map.getPitch(), bearing = map.getBearing();

    map.setStyle(styleDef.style);

    map.once('style.load', () => {
        map.jumpTo({ center, zoom, pitch, bearing });
        switchTab(currentTab);  // 重建 GeoJSON 图层
    });
}
```

### 4.4 点聚合可视化（第11讲挑战1）

**功能描述**：使用 MapLibre 内置的 `cluster` 功能，按缩放级别动态聚合散点。

**聚合配置**：

| 参数 | 值 | 说明 |
|------|:--:|------|
| `cluster` | `true` | 启用聚合 |
| `clusterMaxZoom` | `14` | 超过14级不再聚合 |
| `clusterRadius` | `50` | 聚合半径（像素） |

**三层渲染结构**：

```
earthquakes Source (cluster: true)
    │
    ├── clusters Layer (circle)
    │   filter: ['has', 'point_count']
    │   ├── 圆大小 step: point_count → 20/30/40px
    │   └── 圆颜色 step: point_count → 蓝/黄/粉
    │
    ├── cluster-count Layer (symbol)
    │   filter: ['has', 'point_count']
    │   └── 文字: {point_count_abbreviated}
    │
    └── unclustered-point Layer (circle)
        filter: ['!', ['has', 'point_count']]
        ├── 大小: interpolate mag → 4~20px
        └── 颜色: interpolate mag → 绿→黄→红
```

**点击展开逻辑**：

```javascript
map.on('click', 'clusters', async (e) => {
    const clusterId = features[0].properties.cluster_id;
    const source = map.getSource('earthquakes');
    const zoom = await source.getClusterExpansionZoom(clusterId);
    map.easeTo({ center: features[0].geometry.coordinates, zoom, duration: 600 });
});
```

**数据源**：USGS 地震 GeoJSON (`earthquakes.geojson`)，回退方案为北京地标数据。

### 4.5 3D 建筑拉伸（fill-extrusion）

**功能描述**：使用 `fill-extrusion` 图层类型，将北京 CBD 的 15 栋建筑 Polygon 拉伸为 3D 立体。

**数据源**：自建北京 CBD 建筑 GeoJSON（15 个 Polygon Feature），属性包含 `height`（米）、`floors`（层数）、`area`（万㎡）。

**3D 渲染参数**：

| 属性 | 值 | 说明 |
|------|-----|------|
| `fill-extrusion-height` | `['get', 'height']` | 从数据属性取高度 |
| `fill-extrusion-base` | `0` | 从地面起 |
| `fill-extrusion-color` | YlOrRd 渐变 | 按高度 interpolation |
| `fill-extrusion-opacity` | `0.85` | 半透明 |

**高度-颜色映射**（YlOrRd 顺序色带）：

| 高度范围 | 颜色 | 代表建筑 |
|---------|------|---------|
| 0~100m | `#ffeda0` 浅黄 | 世贸天阶 |
| 100~200m | `#feb24c` 浅橙 | 建外SOHO、万达 |
| 200~300m | `#fc4e2a` 橙红 | 银泰中心、央视 |
| 300~400m | `#e31a1c` 红色 | 国贸三期 |
| 400~528m | `#800026` 暗红 | 中国尊（528m） |

**3D 视角设置**：

```javascript
map.easeTo({
    center: [116.4600, 39.9100],  // CBD 中心
    zoom: 14.5,
    pitch: 60,                     // 俯仰角 60°
    bearing: -30,                  // 旋转 -30°
    duration: 2000                 // 2秒飞行动画
});
```

## 七、功能亮点总结

1. **Style JSON 自定义**：5 套完全不同的底图方案，涵盖明亮/暗黑/卫星/浅色/地形
2. **数据驱动表达式**：`match` + `interpolate` + `step` 三种表达式灵活运用
3. **样式切换保视角**：切换前后 center/zoom/pitch/bearing 不变，用户体验流畅
4. **内置点聚合**：无需第三方插件，cluster source + 3 层渲染 + 点击展开
5. **WebGL 3D 渲染**：fill-extrusion 按高度拉伸 + YlOrRd 渐变 + 60°倾斜视角
6. **Tab 架构**：三个独立可视化模式，互不干扰，切换时自动清理/重建图层
