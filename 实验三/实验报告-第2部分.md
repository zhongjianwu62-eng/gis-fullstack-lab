## 四、分级设色图实现

### 4.1 分级策略

采用**自定义分级**（Manual Breaks），根据中国 GDP 分布特点设置 8 个级别：

| 级别 | GDP范围（亿元） | 含义 | 代表省份 |
|:----:|:-------------:|------|---------|
| 1 | < 10,000 | 极低 | 西藏、青海、宁夏、海南 |
| 2 | 10,000 ~ 20,000 | 低 | 甘肃、吉林、黑龙江、山西 |
| 3 | 20,000 ~ 35,000 | 中低 | 内蒙古、广西、贵州、云南 |
| 4 | 35,000 ~ 55,000 | 中等 | 北京、河北、安徽、福建 |
| 5 | 55,000 ~ 80,000 | 中高 | 河南、湖北、湖南、四川 |
| 6 | 80,000 ~ 100,000 | 高 | 浙江、山东 |
| 7 | 100,000 ~ 135,000 | 极高 | 江苏 |
| 8 | > 135,000 | 最高 | 广东 |

**分级依据**：采用指数型递增间隔（1万→1万→1.5万→2万→2.5万→2万→3.5万），在低值区密集分级以更好区分欠发达省份间的差异，高值区稀疏以突出头部省份。

### 4.2 配色方案（顺序色带）

**使用 ColorBrewer YlOrRd 7级顺序色带**：

| 色阶 | 颜色值 | 视觉亮度 | 含义 |
|:---:|--------|:------:|------|
| L1 | `#FFEDA0` | 高亮度浅黄 | 最低 GDP |
| L2 | `#FED976` | 亮黄 | 很低 |
| L3 | `#FEB24C` | 浅橙 | 较低 |
| L4 | `#FD8D3C` | 橙色 | 中等 |
| L5 | `#FC4E2A` | 橙红 | 较高 |
| L6 | `#E31A1C` | 红色 | 很高 |
| L7 | `#BD0026` | 深红 | 极高 |
| L8 | `#800026` | 暗红 | 最高（广东省专用） |

**配色选择依据**（顺序色带 Sequential）：

1. **数据类型**：GDP 是**连续数值型数据**，从低到高有明确的方向性和递进关系
2. **色带特性**：YlOrRd 色带通过**亮度递减 + 饱和度递增**来编码数值大小：低值用高亮度浅色（浅黄），高值用低亮度深色（深红），符合人眼"深色=更多/更大"的视觉直觉
3. **色觉无障碍**：ColorBrewer YlOrRd 色带经测试在色盲友好性方面表现良好，即使红绿色盲也能通过亮度差区分不同等级
4. **不选用发散色带**：GDP 数据没有自然的"中心点"或正负对称关系，发散色带（如 RdBu 红蓝）适合有零点/均值参考的数据（如增长率 -5%~+5%），此处不合适
5. **不选用定性色带**：定性色带（如 Set1）适合分类数据（如土地利用类型），用于连续数值会造成错误的类别暗示

### 4.3 核心代码

```javascript
// getColor 分级函数
function getGDPColor(gdp) {
    if (gdp === undefined || gdp === null || gdp === 0) return '#cccccc';
    for (let i = GDP_GRADES.length - 1; i >= 0; i--) {
        if (gdp > GDP_GRADES[i]) return GDP_COLORS[i];
    }
    return GDP_COLORS[0];
}

// style 回调
function choroplethStyle(feature) {
    const adcode = feature.properties.adcode;
    const data = PROVINCE_DATA[adcode];
    const gdp = data ? data.gdp : 0;
    return {
        fillColor: getGDPColor(gdp),
        weight: 1,
        opacity: 1,
        color: '#ffffff',    // 白色省界线，降低视觉干扰
        fillOpacity: 0.8
    };
}

// L.geoJSON 创建
choroplethLayer = L.geoJSON(geoJSONData, {
    style: choroplethStyle,
    onEachFeature: onEachProvince
}).addTo(map);
```

## 五、比例符号图实现

### 5.1 符号大小映射

使用**平方根缩放**（Flannery 感知校正）：

$$r = \sqrt{population} \times 1.5$$

**为什么不用线性缩放？**

心理学家 Flannery 的研究表明：人眼在比较圆形面积时存在系统性低估——大脑感知到的面积差异约为实际差异的 0.7 次幂。使用平方根缩放可以对此进行补偿，使得读者感知到的符号大小比例更接近实际数值比例。

如果 $r = population \times k$（线性缩放），人口 2000 万的城市符号面积将是 200 万的 100 倍，完全无法在地图上显示。平方根将比例压缩到 10:1，视觉上可区分。

### 5.2 pointToLayer 实现

```javascript
// pointToLayer：将 GeoJSON Point → CircleMarker
function pointToLayer(feature, latlng) {
    const pop = feature.properties.population;
    const radius = getSymbolRadius(pop);
    return L.circleMarker(latlng, {
        radius: radius,
        fillColor: '#3388ff',
        color: '#1a5dc7',
        weight: 2,
        opacity: 0.9,
        fillOpacity: 0.55
    });
}

// 使用 L.geoJSON 的 pointToLayer 选项
symbolLayer = L.geoJSON(citiesGeoJSON, {
    pointToLayer: pointToLayer,
    onEachFeature: onEachCity
}).addTo(map);
```

### 5.3 符号样式选择

- **填充色**：`#3388ff`（标准蓝），与 YlOrRd 色带的暖色调形成**冷暖对比**，两种专题图层叠加时互不干扰
- **边框**：`#1a5dc7` 深蓝 2px，确保小符号也可见
- **透明度**：`fillOpacity: 0.55`，允许底层分级设色面透过圆形符号显示

## 六、filter 过滤功能

实现了**模式切换**功能，通过 4 个按钮切换显示模式：

| 模式 | 实现方式 | 效果 |
|------|---------|------|
| 📊 全部显示 | 默认渲染 | 同时显示分级设色面 + 比例符号圆 |
| 🎨 仅分级设色 | 移除 symbolLayer | 仅显示省级 GDP 色块 |
| 🔵 仅比例符号 | 移除 choroplethLayer | 仅显示城市人口圆形符号 |
| 💰 高GDP省份 | `filter: filterHighGDP` | 仅显示 GDP ≥ 50000 亿的省份 |

**filter 函数实现**：

```javascript
function filterHighGDP(feature) {
    const adcode = feature.properties.adcode;
    const data = PROVINCE_DATA[adcode];
    return data && data.gdp >= 50000;
}

// 在 L.geoJSON 中应用
L.geoJSON(geoJSONData, {
    style: choroplethStyle,
    onEachFeature: onEachProvince,
    filter: filterHighGDP  // 仅渲染符合条件的要素
}).addTo(map);
```

## 十一、配色方案选择依据总结

### 为什么选择顺序色带（Sequential YlOrRd）？

| 色带类型 | 适用场景 | 本实验是否适用 |
|---------|---------|:----------:|
| **顺序型 Sequential** | 连续数值，有低→高方向 | ✅ **适用** — GDP 是连续数值 |
| **发散型 Diverging** | 有中心值/零点的数据 | ❌ 不适用 — GDP 没有自然中心 |
| **定性型 Qualitative** | 分类/名义数据 | ❌ 不适用 — 会暗示省级间是类别关系 |

### YlOrRd 色带的视觉层次

```
低GDP                             高GDP
浅黄 ──→ 亮黄 ──→ 浅橙 ──→ 橙 ──→ 橙红 ──→ 红 ──→ 深红 ──→ 暗红
高亮度                           低亮度
低饱和度                         高饱和度
"少/小"                          "多/大"
```

人眼天然将**深色/高饱和度**解读为"更多/更大"，将**浅色/低饱和度**解读为"更少/更小"，这种映射符合视觉认知直觉，无需额外学习即可理解。
