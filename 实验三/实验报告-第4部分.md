## 四、数据内容设计

### 4.1 主题选择

采用第12讲推荐的 **"我的人生轨迹"** 主题，原因是：

1. **数据真实**：17个地点全部基于个人真实经历，有情感价值
2. **空间覆盖广**：横跨6省1市（广东、湖北、北京、上海、陕西、四川），地图展示效果好
3. **时间跨度长**：2002~2025共计23年，适合时间线叙事
4. **分类明确**：5个分类覆盖人生主要维度，且视觉上容易区分

### 4.2 数据字段设计

每个 POI Feature 包含以下属性：

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `id` | number | 唯一标识 | `1` |
| `name` | string | 地点名称 | `"北京故宫"` |
| `category` | string | 分类标识 | `"travel"` |
| `categoryName` | string | 分类中文名 | `"旅行足迹"` |
| `date` | string | 起始日期 (YYYY-MM) | `"2022-07-20"` |
| `endDate` | string | 结束日期（可选） | `"2025-06"` |
| `description` | string | 地点描述 | `"故宫的中轴线..."` |
| `icon` | string | emoji 图标 | `"🏯"` |
| `importance` | number | 重要程度 (0-100) | `92` |
| `memory` | string | 特别记忆（可选） | `"用手机GPS..."` |
| `city` | string | 所在城市 | `"北京"` |
| `province` | string | 所在省份 | `"北京市"` |

| 卡片 | 白色 + `12px` 圆角 + 三级阴影 | 层次清晰，视觉统一 |
| 字体 | 系统字体栈 | PingFang SC / Microsoft YaHei，零额外加载 |
| 间距 | 12px/16px/24px | 基于 8px 网格 |
| 动画 | 200ms cubic-bezier | 流畅不突兀 |

### 5.2 响应式设计

| 断点 | 布局 | 侧边栏 | 地图 |
|------|------|:---:|------|
| ≥900px（桌面） | 左右并排 | 320px 固定 | flex:1 填充 |
| <900px（平板） | 上下堆叠 | 100% 宽，max-height 40vh | 固定 60vh |
| <480px（手机） | 紧凑间距 | padding/gap 缩减 | 固定 50vh |

### 5.3 交互一致性

所有可交互元素遵循统一的交互模式：

1. **可点击元素** → hover 时颜色变化 + 微位移 + 阴影变化
2. **选中状态** → 高亮色 + 放大 + 脉冲动画
3. **加载中** → 网络请求失败时显示错误提示，提供重试
4. **数据为空** → 显示友好的占位文案（"暂无书签，点击下方按钮添加"）

## 六、功能实现详解

### 6.1 多图层管理

**底图切换**：通过 `L.control.layers` 管理 5 套底图：

| 底图 | 类型 | 用途 |
|------|------|------|
| OSM 标准 | 栅格瓦片 | 默认底图，信息适中 |
| CartoDB 浅色 | 栅格瓦片 | 专题数据展示时使用 |
| ESRI 卫星 | 栅格瓦片 | 查看真实地貌 |
| CartoDB 暗黑 | 栅格瓦片 | 暗色主题 |
| 天地图 | WMTS | 国内详细数据（需 Key） |

POI 标记层和 GDP 专题层通过 `L.layerGroup` 独立管理，不受底图切换影响。

### 6.2 三种探索模式

**模式切换核心代码**：

```javascript
function switchMode(mode) {
  currentMode = mode;
  if (measureActive) stopMeasure();

  switch (mode) {
    case 'explore':
      clearTimelineLine();
      clearChoropleth();
      hideGDPLegend();
      loadPOIs();
      break;
    case 'timeline':
      clearChoropleth();
      hideGDPLegend();
      loadPOIs();
      drawTimeline();   // 按时间排序 + 蓝色虚线连接
      break;
    case 'thematic':
      clearTimelineLine();
      loadPOIs();
      loadChoropleth();  // 异步加载 DataV GeoJSON + 分级设色
      break;
  }
}
```

**时间线模式**：按 `date` 字段排序 → 提取坐标数组 → `L.polyline` 蓝色虚线连接 → 每个点标注顺序数字。

**专题模式**：异步 fetch 阿里 DataV API → `L.geoJSON` 渲染 → `getGDPColor` 分级设色 → `onEachFeature` 绑定 hover + click 交互。

### 6.3 GDP 分级设色（整合第10讲能力）

**配色方案**：ColorBrewer YlOrRd 8 级顺序色带

**选择依据**（来自第10讲理论）：

| 色带类型 | 适用场景 | 本场景是否适用 |
|---------|---------|:----------:|
| **顺序型 Sequential** | 连续数值，有低→高方向 | ✅ GDP 是连续数值 |
| **发散型 Diverging** | 有中心值/零点 | ❌ GDP 无自然中心 |
| **定性型 Qualitative** | 分类/名义数据 | ❌ 省份间是数值关系 |

**分级策略**（自定义断点，指数递增）：

| 级别 | GDP范围（亿元） | 颜色 | 代表省份 |
|:----:|:-------------:|------|---------|
| 1 | < 5,000 | `#FFEDA0` | 西藏、青海、宁夏 |
| 2 | 5,000 ~ 10,000 | `#FED976` | 海南、甘肃 |
| 3 | 10,000 ~ 20,000 | `#FEB24C` | 吉林、黑龙江、山西 |
| 4 | 20,000 ~ 35,000 | `#FD8D3C` | 内蒙古、广西、贵州 |
| 5 | 35,000 ~ 55,000 | `#FC4E2A` | 北京、河北、安徽 |
| 6 | 55,000 ~ 80,000 | `#E31A1C` | 河南、湖北、四川 |
| 7 | 80,000 ~ 100,000 | `#BD0026` | 浙江、山东 |
| 8 | > 100,000 | `#800026` | 江苏、广东 |

### 6.4 分类 POI 标记（整合第09讲能力）

**DivIcon 工厂函数**：

```javascript
function createCustomIcon(emoji, category) {
  const color = categoryConfig[category].color;
  return L.divIcon({
    html: `<div style="width:40px;height:40px;background:${color};
      border-radius:50%;display:flex;align-items:center;justify-content:center;
      font-size:20px;border:3px solid #fff;box-shadow:0 3px 12px rgba(0,0,0,0.25);">
      ${emoji}</div>`,
    iconSize: [40, 40],
    iconAnchor: [20, 20],
    popupAnchor: [0, -22]
  });
}
```

**高亮状态**（选中时）：图标放大到 52px + 脉冲 CSS 动画 + 阴影加深。

**Popup 模板**：渐变色头部（与分类色统一）+ 日期 + 描述 + 回忆引用 + 分类标签。

### 6.5 距离测量（整合第09讲练习2）

- **激活**：点击 `📏 测距` 按钮或 `Shift + 点击地图`
- **算法**：Haversine 球面距离公式
- **显示**：每段线段标注距离，底部累计总距离
- **视觉**：红色虚线 + 红色圆点标记 + 白色文字标签

### 6.6 书签系统（整合第09讲挑战）

- **存储**：`localStorage` 持久化，JSON 序列化
- **结构**：`[{ name, lat, lng, zoom, time }]`
- **功能**：添加（当前视图）/ 点击飞行 / 删除

### 6.7 MapLibre 高级功能入口

通过头部两个导航按钮跳转到独立页面：
- `🏗️ 3D视图` → `maplibre.html`（第3部分）
- `📊 专题图` → `专题地图.html`（第2部分）

这种"主页 + 子页"的架构体现了模块化设计思想——核心功能在主页（Leaflet），高级可视化在子页（MapLibre），既保证了主页的加载速度和兼容性，又展示了多引擎切换能力。

### 7.2 设计原则

1. **数据与逻辑分离**：`data.js` 只存数据，`config.js` 只存工具函数，`main.js` 只处理交互
2. **配置驱动**：分类配置（`categoryConfig`）和模式配置（`modeConfig`）集中管理，添加新分类无需修改 UI 代码
3. **状态管理**：全局状态（`currentCategory`, `currentMode`, `measureActive`）显式声明在文件顶部

| 技术点 | 说明 |
|--------|------|

function saveEditorPOI() {
    const feature = {
        type: 'Feature',
        properties: { id: Date.now(), name, category, ... },
        geometry: { type: 'Point', coordinates: [lng, lat] }
    };
    const customs = getCustomPOIs();
    customs.push(feature);
    saveCustomPOIs(customs);  // localStorage 持久化
    loadPOIs();                // 刷新地图
    updatePlacesList();        // 刷新列表
    initStats();               // 刷新统计
}
