## 一、原理篇（Principle）【约15分钟】

### 1.1 为什么需要搭建专业的开发环境？

想象一下，你准备做一道菜。如果厨房里没有合适的锅碗瓢盆、没有调料、没有切菜板，即使你厨艺再好，也很难做出美味的菜肴。同样的道理，**开发环境就是程序员的"厨房"**——它决定了你能否顺利完成开发工作。

**为什么不能"边学边装"？**

很多同学可能会想："我遇到需要的时候再安装不行吗？"这种方式存在几个问题：

1. **版本冲突**：不同工具之间可能存在版本依赖关系，临时安装容易造成兼容性问题
2. **环境污染**：多次安装卸载会在系统中留下"垃圾"，导致各种奇怪的错误
3. **效率低下**：每次遇到新需求都要停下来配置环境，打断学习和开发的连续性
4. **难以复现**：当你的代码在同学电脑上跑不起来时，很难排查是代码问题还是环境问题

因此，在课程开始时**一次性搭建好完整的开发环境**，是高效学习的基础。

### 1.2 核心概念

在开始安装之前，我们需要理解几个核心概念：

#### 1.2.1 IDE（集成开发环境）

**IDE（Integrated Development Environment）** 是程序员编写代码的主要工具。它不仅仅是一个文本编辑器，还集成了：

- **代码编辑器**：语法高亮、自动补全、格式化
- **调试器**：断点调试、变量监视
- **版本控制**：Git集成
- **终端**：命令行操作
- **插件系统**：扩展功能

本课程我们主要使用两款IDE：

| IDE | 定位 | 特点 |
|-----|------|------|
| **VS Code** | 通用代码编辑器 | 轻量、插件丰富、免费开源 |
| **Cursor** | AI原生IDE | 基于VS Code、深度集成AI、智能编程 |

#### 1.2.2 运行时环境（Runtime）

代码写完后需要"运行"，而不同的编程语言需要不同的运行时环境：

| 语言 | 运行时 | 本课程版本 | 作用 |
|------|--------|-----------|------|
| JavaScript | Node.js | 22.x LTS | 运行前端构建工具、开发服务器 |
| Python | Python解释器 | 3.12+ | 运行空间数据处理脚本、AI模型 |
| Java | JDK（Java Development Kit） | 21 LTS | 运行Spring Boot后端服务 |

#### 1.2.3 包管理器（Package Manager）

包管理器用于安装和管理项目依赖的第三方库：

| 语言 | 包管理器 | 功能 |
|------|---------|------|
| JavaScript | npm / pnpm | 安装Vue、Cesium等前端库 |
| Python | pip / conda | 安装GeoPandas、Rasterio等库 |
| Java | Maven / Gradle | 管理Spring Boot、GeoTools等依赖 |

#### 1.2.4 环境变量（Environment Variables）

环境变量是操作系统中的全局设置，告诉系统去哪里找可执行程序。最重要的环境变量是**PATH**——它包含一系列目录路径，当你在终端输入命令时，系统会按顺序在这些目录中查找对应的程序。

```
举例：当你输入 "node --version" 命令时
```

### 1.4 技术对比：传统IDE vs AI原生IDE

2024年以来，AI编程助手已经成为开发者的标配工具。本课程推荐使用**Cursor**作为主力IDE：

| 对比维度 | VS Code + Copilot | Cursor |
|---------|------------------|--------|
| **AI集成深度** | 插件形式，功能受限 | 原生集成，深度优化 |
| **代码补全** | 单行/多行补全 | 整文件级补全 |
| **对话能力** | 独立面板 | 内联对话、@引用上下文 |
| **代码修改** | 需手动复制粘贴 | 一键应用到代码 |
| **学习曲线** | 较平缓 | 略陡（但值得投入） |
| **价格** | Copilot需付费 | 有免费额度 |

**本课程策略**：优先使用Cursor，同时熟悉VS Code（因为很多企业仍在使用）。

Cursor的核心功能：
- **Tab补全**：按Tab键接受AI生成的代码
- **Cmd+K（Ctrl+K）**：内联编辑，选中代码后用自然语言描述修改需求
- **Cmd+L（Ctrl+L）**：打开AI对话面板
- **@符号引用**：在对话中引用文件、文档、代码片段

#### 2.2.3 Node.js 生态

Node.js是JavaScript的运行时环境，在GIS开发中主要用于：

1. **运行前端构建工具**：Vite、Webpack等
2. **开发服务器**：热更新、代理API请求
3. **命令行工具**：各种CLI工具

**npm vs pnpm**：
- **npm**：Node.js默认包管理器，使用最广泛
- **pnpm**：更快、更省磁盘空间的替代方案（推荐）

```
npm 安装依赖的方式（每个项目独立副本）：

项目A/node_modules/lodash/
项目B/node_modules/lodash/  ← 重复！
项目C/node_modules/lodash/  ← 重复！

pnpm 的方式（全局存储 + 硬链接）：

~/.pnpm-store/lodash/
项目A/node_modules/lodash → 链接
项目B/node_modules/lodash → 链接
项目C/node_modules/lodash → 链接

结果：节省大量磁盘空间，安装速度更快
```

#### 2.2.4 Python 环境管理

Python在GIS开发中主要用于空间数据处理和AI模型推理。由于不同项目可能需要不同版本的库，我们需要使用**虚拟环境**来隔离：

| 工具 | 特点 | 推荐场景 |
|------|------|---------|
| **venv** | Python内置、轻量 | 简单项目、教学环境 |
| **conda** | 功能强大、支持非Python包 | 科学计算、需要GDAL等复杂依赖 |

本课程推荐使用**venv**作为基础，遇到复杂依赖时再考虑conda。

#### 2.2.5 JDK与Java生态

JDK 21是当前的长期支持版本（LTS），带来了重要的新特性：

- **虚拟线程（Virtual Threads）**：大幅提升并发性能
- **模式匹配**：更简洁的代码
- **序列集合**：新的集合API

在GIS开发中，Java主要用于后端服务开发，配合Spring Boot框架。

### 3.1 Windows系统安装步骤

#### 3.1.1 安装VS Code

1. 访问 https://code.visualstudio.com/
2. 点击"Download for Windows"
3. 运行安装程序，**勾选以下选项**：
   - ✅ 添加"通过Code打开"操作到Windows资源管理器文件上下文菜单
   - ✅ 添加"通过Code打开"操作到Windows资源管理器目录上下文菜单
   - ✅ 将Code注册为受支持的文件类型的编辑器
   - ✅ 添加到PATH

#### 3.1.2 安装Cursor

1. 访问 https://cursor.sh/
2. 点击"Download for Windows"
3. 运行安装程序
4. 首次启动时：
   - 使用邮箱注册账号（或使用GitHub登录）
   - 选择"Import VS Code Settings"导入VS Code配置
   - 选择AI模型（推荐Claude或GPT-4）

#### 3.1.3 安装Node.js 22 LTS

1. 访问 https://nodejs.org/
2. 下载"22.x.x LTS"版本
3. 运行安装程序，使用默认选项
4. 验证安装（打开命令提示符）：

```bash
# 验证Node.js版本
node --version
# 应该输出：v22.x.x

# 验证npm版本
npm --version
# 应该输出：10.x.x
```

5. 配置npm镜像（加速下载）：

```bash
# 设置淘宝镜像
npm config set registry https://registry.npmmirror.com

# 验证配置
npm config get registry
```

6. 安装pnpm（推荐）：

```bash
npm install -g pnpm

# 验证pnpm
pnpm --version
```

#### 3.1.4 安装Python 3.12+

1. 访问 https://www.python.org/downloads/
2. 下载Python 3.12.x
3. 运行安装程序，**务必勾选**：
   - ✅ Add python.exe to PATH
   - 选择"Customize installation"
   - ✅ pip
   - ✅ py launcher
4. 验证安装：

```bash
# 验证Python版本
python --version
# 应该输出：Python 3.12.x

# 验证pip
pip --version
```

5. 配置pip镜像：

```bash
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

6. 创建虚拟环境（示例）：

```bash
# 创建项目目录
mkdir gis-project
cd gis-project

# 创建虚拟环境
python -m venv .venv

# 激活虚拟环境（Windows）
.venv\Scripts\activate

# 激活后命令行前会出现 (.venv) 前缀
```

#### 3.1.5 安装JDK 21 LTS

1. 访问 https://adoptium.net/
2. 选择"Temurin 21 (LTS)"，Windows x64版本
3. 下载并运行MSI安装程序
4. 在安装过程中，勾选：
   - ✅ Set JAVA_HOME variable
   - ✅ Add to PATH
5. 验证安装：

```bash
# 验证Java版本
java --version
# 应该输出：openjdk 21.x.x

# 验证javac（编译器）
javac --version
```

如果命令未找到，需要手动配置环境变量：

1. 搜索"环境变量" → "编辑系统环境变量"
2. 点击"环境变量"
3. 在"系统变量"中新建：
   - 变量名：`JAVA_HOME`
   - 变量值：`C:\Program Files\Eclipse Adoptium\jdk-21.x.x-hotspot`（实际安装路径）
4. 编辑"Path"变量，添加：`%JAVA_HOME%\bin`

### 3.2 macOS系统安装步骤

macOS推荐使用**Homebrew**包管理器来安装开发工具：

### 3.5 常见错误与解决

#### 错误1：'node' 不是内部或外部命令

**原因**：Node.js未添加到PATH环境变量

**解决方法**：
1. 重新运行Node.js安装程序，确保勾选"Add to PATH"
2. 或手动添加环境变量：将`C:\Program Files\nodejs`添加到PATH

#### 错误2：pip安装包时权限错误

**原因**：尝试在系统Python目录安装包

**解决方法**：
```bash
# 使用--user标志安装到用户目录
pip install --user package_name

# 或使用虚拟环境（推荐）
python -m venv .venv
.venv\Scripts\activate
pip install package_name
```

#### 错误3：VS Code终端无法识别命令

**原因**：终端使用了PowerShell限制执行策略

**解决方法**：
```powershell
# 以管理员身份运行PowerShell，执行：
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

#### 错误4：JAVA_HOME未设置

**原因**：JDK安装时未自动配置环境变量

**解决方法**：
1. 找到JDK安装目录（如`C:\Program Files\Eclipse Adoptium\jdk-21.0.2+13`）
2. 新建系统环境变量`JAVA_HOME`，值为上述目录
3. 在PATH中添加`%JAVA_HOME%\bin`
4. 重启命令行窗口

#### 错误5：pnpm安装失败

**原因**：网络问题或npm版本过旧

**解决方法**：
```bash
# 先更新npm
npm install -g npm@latest

# 使用淘宝镜像安装pnpm
npm install -g pnpm --registry=https://registry.npmmirror.com
```
