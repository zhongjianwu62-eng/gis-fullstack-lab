## 子任务 ①：编写环境自检脚本

### 目标

一键验证 Node.js / Python / Java / Git 四个工具链的安装状态与版本号。

### 操作步骤

1. **新建 `check_env.bat`**（Windows 批处理脚本）：
   - 使用 `chcp 65001` 设置 UTF-8 编码避免中文乱码
   - 依次用 `where` 检测工具路径，用 `-v` / `--version` 获取版本号
   - 未安装的工具输出 `[WARN]` 警告
   - 结尾 `pause` 保持窗口不自动关闭

2. **新建 `check_env.sh`**（Unix / Git Bash 脚本）：
   - 使用 `command -v` 检测工具路径
   - 兼容 `python` 和 `python3` 两种命名
   - 使用 `set -e` 确保遇错即停

3. **运行验证**：
   ```bash
   # Windows
   check_env.bat

   # Git Bash
   bash check_env.sh
   ```

## 子任务 ②：搭建三层目录骨架

### 目标

搭建「前端展示层 + Python 数据处理层 + Java 后端服务层」三层全栈目录结构。

### 操作步骤

1. **创建目录结构**：
   ```bash
   mkdir python-data
   mkdir -p java-backend/src/main/java/com/gis
   ```

2. **Python 数据处理层**（`python-data/`）：
   - `data_processor.py`：GeoJSON 读写、坐标转换、边界框生成
   - `requirements.txt`：geopandas / shapely / fiona / pyproj / folium

3. **Java 后端服务层**（`java-backend/`）：
   - `pom.xml`：Spring Boot 3.4 + JTS 1.19 空间拓扑库
   - `GisApplication.java`：Spring Boot 入口类

4. **纳入 Git 版本控制**：
   ```bash
   git add python-data/ java-backend/ check_env.bat check_env.sh
   git commit -m "feat: 搭建三层全栈目录骨架与环境自检脚本"
   ```

## 子任务 ④：各层在 GIS 工程中的职责分工

### 三层职责说明

| 层级 | 核心技术 | 在 GIS 工程中的具体职责 |
|:----:|----------|------------------------|
| **前端展示层** | Vue 3 + Leaflet + TypeScript | ① 渲染瓦片底图（高德/OSM）② 展示空间要素（点/线/面标注）③ 处理用户交互（缩放/拖拽/点击查询）④ 实现 GIS UI 功能（城市快跳、标注管理） |
| **数据处理层** | Python + GeoPandas + Shapely + Fiona | ① 读取矢量数据（Shapefile/GeoJSON/KML）② 坐标系转换（WGS84↔GCJ02↔BD09）③ 缓冲区分析、叠加分析④ 空间数据清洗与格式转换⑤ 为前后端提供预处理 GeoJSON |
| **后端服务层** | Java · Spring Boot + JTS + PostGIS | ① RESTful API 对外提供空间查询接口② JTS 实现空间拓扑运算（相交/包含/距离）③ 管理空间数据库（PostGIS）④ 属性查图 / 空间查图⑤ 用户认证与操作日志 |

### 协作流程

```
用户浏览器
  │
  ▼
[前端展示层]  ← 渲染地图、响应用户操作
  │ GET /api/features?bbox=...
  ▼
[后端服务层]  ← 接收请求、空间查询、返回 GeoJSON
  │ SELECT ... FROM spatial_table WHERE ST_Intersects(...)
  ▼
[PostGIS 数据库]  ← 存储矢量数据、执行空间索引查询
  │ 预处理数据写入
  ▼
[数据处理层]  ← 定期 ETL：清洗、转换、入库
```
